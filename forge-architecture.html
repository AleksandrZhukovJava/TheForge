Что это

Инженерная платформа, где LLM не видно

Разработчик не выбирает модель, не пишет промпт, не ищет нужный Merge Request и не думает, какой API вызвать. Он запускает Skill — «Review this Merge Request», «Explain this Service», «Implement this Jira Story» — а всё остальное берёт на себя платформа. Каждое действие сначала пытается выполниться детерминированно; интеллект подключается только там, где он действительно нужен.

Skill First

Архитектура

Ядро, оркестратор и исполнители

Пользователь запускает Skill из каталога, оркестратор интерпретирует его Recipe и для каждого Strike формует Stock, выбирает исполнителя и фиксирует результат. Вся эта машина стоит на инфраструктурном фундаменте.

Исполнители · Automation First

Чем ниже падаешь — тем горячее

Для каждого Strike Forge Brain перебирает исполнителей строго по приоритету. Пока задачу можно решить детерминированно, платформа остаётся «холодной». Smith — это огонь кузницы: интеллект, к которому тянутся в последнюю очередь.

// Резолвер Forge Brain — «правило выбора» как код, а не как картинка
fun resolve(step, ctx):
  candidates = registry.providersFor(step.capability)
          .filter { it.canHandle(step, ctx) }
  if candidates.isEmpty(): throw NoExecutorFound(step)
  // TOOL(0) < PRESS(1) < MASTER(2) < SMITH(3) — это и есть Automation First
  return candidates.sortedBy(kind.ordinal, -priority).first()

Промежуточный слой — Capability Registry: Step объявляет capability, плагины регистрируют, что её умеют. Без реестра «правило выбора» из ТЗ нереализуемо. Жёсткий запрет (напр. GitLab merge / force-push) означает, что capability просто не зарегистрирована — резолвить нечего.

Anvil

Сборка контекста до выполнения

Перед любым Strike именно Anvil формует нужную информацию в единый Stock — лениво и параллельно, только то, что Strike объявил в своём запросе. Тяжёлое модель дотягивает вызовами Skills, а не RAG-стаффингом. Подсвечены источники MVP.

Git GitLab Jira Confluence локальный проект терминал Docker Kubernetes PostgreSQL Redis Forge Memory результаты прошлых Strike'ов

◆ — источники MVP (Git · GitLab · Jira · Confluence). Остальные подключаются как Context Provider-плагины без изменения ядра.

Кодовая база

Модульная карта

Всё зависит только от forge-sdk. Плагины не знают про ядро — лишь про контракты. Это и делает «Everything is a Plugin» реальным, а не лозунгом.

the-forge/ ├─ forge-sdk/ # публичные контракты. только интерфейсы. нулевые зависимости ├─ forge-core/ # ядро: plugins · DI · config · events · security · state · audit ├─ forge-brain/ # оркестратор: RecipeEngine · StrikeResolver · ModelRouter ├─ forge-anvil/ # движок контекста + встроенные ContextProvider'ы ├─ forge-memory/ # MemoryProvider SPI + хранилище (SQLite + vector) ├─ forge-executors/ # рантаймы Tool / Press / Master / Smith ├─ plugins/ │ ├─ integration-gitlab/ # Tool + ContextProvider (БЕЗ merge / force-push) │ ├─ integration-jira/ # Tool + ContextProvider │ ├─ integration-confluence/ │ ├─ integration-git/ │ └─ smith-claude-code/ # первый Smith — CLI-агент ├─ skills/ # review-pr · explain-service · generate-liquibase · … ├─ forge-workshop/ # Compose Desktop приложение (сборка всего) └─ forge-cli/ # v0.3 — CLI поверх того же ядра

Шесть решений

Спорные развилки закрыты, чтобы дизайн ядра не переделывался позже.

Q1
Smith: роли моделей
Qwen — текст, Claude Code — код/ревью
Единый контракт Smith. Роутинг по роли задачи: локальная Qwen продумывает текст и планирует, агент пишет код и ревью.
Q2
Dependency Injection
Koin
Лёгкий, kotlin-идиоматичный, без annotation processing. Spring для десктопа избыточен.
Q3
Изоляция плагинов
SPI под изоляцию, включаем ≈P7
P0 — общий classloader; отдельный classloader на плагин и жёсткие permissions — перед публичным SDK.
Q4
Модель Recipe
Граф (DAG) с самого начала
Линейный — вырожденный DAG. Готово к ветвлениям, ошибкам и Workflow Designer без миграции ядра.
Q5
Формат Skill
Гибрид: дата-модель + Kotlin DSL
DSL даёт типобезопасность first-party Skills; сериализуемая дата открывает Marketplace.
Q6
Дистрибуция
Форматы .msi / .dmg; инструмент — позже
Кандидат — Conveyor (подпись + notarization + дельта-апдейты). Дизайн от выбора не зависит.
